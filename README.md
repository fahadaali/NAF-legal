# منصة «مستشار ناف» ⚖️

منصّة استشارات قانونية ذكية باللغة العربية تُنتج **مسوّدات عمل** (صحيفة دعوى، مذكرة رد، عقد، لائحة، رأي قانوني…) بالاعتماد على قاعدة معرفة نظامية سعودية (RAG) ومحرّك توليدي (Claude)، مبنيّة بالكامل على منظومة Cloudflare.

> **مبدأ حاكم:** كل مخرجات المنصّة مسوّدات مساعِدة تخضع لمراجعة محامٍ بشري قبل الاعتماد.

هذا المستودع هو تنفيذ وثيقة المواصفات `nafadvisorplatformspec.md` عبر مراحلها السبع.

---

## نظرة معمارية

```
                         ┌───────────────────────────┐
   المتصفّح (RTL SPA) ───▶│  Cloudflare Worker (Hono) │
   React + Vite          │  المصادقة · التوجيه · SSE  │
                         └────────────┬──────────────┘
                                      │
        ┌──────────────┬─────────────┼──────────────┬───────────────┐
        ▼              ▼             ▼               ▼               ▼
      D1            R2          Vectorize        Workers AI      Claude API
   (بيانات)      (ملفات)      (RAG متجهي)     (تضمين bge-m3)   (تخطيط/توليد
                                                                + web_search)
        ▲                                                            
        │  Queue (تضمين دفعي) · Cron (تتبّع الأنظمة) · KV (جلسات/حدود المعدّل)
```

**تدفّق المحادثة (§5):** رسالة المستخدم → **المُخطِّط** (نموذج خفيف يُنتج خطة JSON) → **المُنفِّذ** (استرجاع RAG + بحث إنترنت + قراءة المرفقات) → تجميع البرومبت مع توجيه النوع → **التوليد المتدفّق** (نموذج قوي) → تنسيق وتصدير.

## البنية على القرص

```
src/                     # خلفية Cloudflare Worker
  index.ts               # نقطة الدخول: التوجيه، الأمان، Cron، Queue
  types.ts               # أنواع البيئة (bindings)
  routes/                # auth · conversations · chat · files · kb · admin
  lib/                   # crypto · auth · claude · planner · rag · prompts · docx · extract
  ingest.ts              # مستهلك Queue: التقسيم والتضمين إلى Vectorize
  cron.ts                # تتبّع الأنظمة عبر بحث مقيّد بالمصادر الرسمية
migrations/              # مخطط D1 (§9) + بذور الأنظمة الأولية
web/                     # واجهة React + Vite (RTL)
  src/components/        # Auth · Sidebar · ChatView · Admin
  src/lib/               # api · consultations · markdown
wrangler.toml            # bindings: D1, R2, Vectorize, AI, KV, Queue, Cron
```

## التشغيل محليًا

```bash
# 1) تثبيت الاعتماديات
npm install
npm --prefix web install

# 2) إعداد الأسرار المحلية
cp .dev.vars.example .dev.vars   # ثم عدّل ANTHROPIC_API_KEY و JWT_SECRET

# 3) تهيئة قاعدة البيانات المحلية
npm run db:migrate
npm run seed                     # اختياري: بذر قائمة الأنظمة الأولية

# 4) بناء الواجهة ثم تشغيل الـ Worker (يقدّم الواجهة + الـ API)
npm run build:web
npm run dev                      # http://localhost:8787
```

للتطوير مع إعادة التحميل الحيّ للواجهة: شغّل `npm run dev` (Worker على 8787) و`npm run dev:web` (Vite على 5173 مع وكيل للـ API) في طرفيتين.

> **ملاحظة:** `Vectorize` و`Workers AI` لا تعملان في وضع `--local` (يظهران "not supported")؛ لاختبار RAG والتضمين فعليًا استخدم `wrangler dev --remote` أو النشر. تعمل المصادقة والمحادثات ورفع الملفات والتصدير محليًا.

## النشر إلى Cloudflare

```bash
# أنشئ الموارد مرّة واحدة، وحدّث المعرّفات في wrangler.toml
wrangler d1 create naf_legal
wrangler r2 bucket create naf-legal-files
wrangler vectorize create naf-legal-kb --dimensions=1024 --metric=cosine
wrangler kv namespace create KV
wrangler queues create naf-legal-ingest

# أسرار الإنتاج
wrangler secret put ANTHROPIC_API_KEY
wrangler secret put JWT_SECRET

# الترحيل والنشر
npm run db:migrate:remote
npm run deploy
```

> بُعد المتجهات `1024` مطابق لنموذج `@cf/baai/bge-m3`.

**أول مستخدم يُسجَّل يصبح مسؤولًا تلقائيًا** (يملك لوحة الإدارة).

## القرارات المتّخذة (§14)

قابلة للتعديل — تغييرها موضعي:

1. **المصادقة:** JWT مخصّص مُخزَّن كـ `httpOnly cookie` مع تجزئة `PBKDF2-SHA256` (متاح أصلًا في Workers دون اعتماديات). يمكن إضافة `Cloudflare Access` للمسؤولين لاحقًا.
2. **إطار الواجهة:** `Vite + React` (SPA خفيف) — أبسط وأخف على Workers Assets من Next.js.
3. **نطاق البحث في الإنترنت:** المستخدم يتحكّم عبر مُبدّل؛ عند تفعيله يدويًا يكون مفتوحًا، وفي مهام تتبّع الأنظمة (Cron) يُقيَّد تلقائيًا بالمصادر الرسمية عبر `allowed_domains`.
4. **طبقات النماذج:** المُخطِّط والتصنيف والاستخراج على `Haiku`؛ التوليد النهائي على `Sonnet` (يُضبطان في `[vars]` بـ `wrangler.toml`). حدّ معدّل 60 طلب/دقيقة لكل مستخدم عبر KV.
5. **قائمة الأنظمة الأولية:** المرافعات الشرعية · المعاملات المدنية · العمل · الشركات · الإثبات · الجمعيات (تُبذر عبر `npm run seed`، ثم يرفع المسؤول الوثائق الرسمية).
6. **توطين البيانات:** `DATA_REGION` في `wrangler.toml`؛ راجِع خيارات `Jurisdictional Restrictions` في D1/R2 لأي اشتراط بالتوطين.

## الأمان والخصوصية (§12)

- **العزل:** كل استعلام على المحادثات والملفّات مقيّد بـ `user_id`؛ لا يرى مستخدم بيانات غيره.
- **سجل التدقيق:** كل فعل إداري (رفع/حذف نظام، تغيير دور، اعتماد تحديث) يُسجَّل مع الفاعل والوقت.
- **حدود المعدّل:** عبر KV لمنع الإساءة والتحكّم في التكلفة.
- **ترويسات أمان:** عبر `secureHeaders` من Hono.
- **إخلاء مسؤولية ثابت:** في واجهة كل محادثة وفي تذييل كل مخرَج Word.
- **PDPL:** التشفير في النقل والتخزين مُفعَّل افتراضيًا على Cloudflare؛ لتوطين البيانات راجِع القيود الاختصاصية لـ D1/R2. الأسرار عبر `wrangler secret` لا في المستودع.

## المخرجات القابلة للتصدير

- **Word (.docx):** يُولَّد أصلًا داخل الـ Worker (كاتب ZIP + CRC32 مضمّن) بتنسيق عربي RTL، خط معقّد، وتذييل إخلاء المسؤولية — دون اعتماديات خارجية.
- **نص (.txt):** نسخ مباشر أو تنزيل.
